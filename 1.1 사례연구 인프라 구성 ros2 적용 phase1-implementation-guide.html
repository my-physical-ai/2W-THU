A. 네트워크 사전 준비 (LeKiwi ↔ NUC 연결)

🛣️
길을 먼저 만들어야 차가 다닌다

rosbridge, ROSboard, 등록 서버를 아무리 잘 설치해도 로봇과 NUC 사이에 길(네트워크)이 없으면 소용없습니다.
우리 환경에서는 ASUS 라우터로 만든 독립 로봇망(10.10.1.x)이 그 길입니다.
자세한 네트워크 셋업은 lekiwi-network-rtsp-edu.html을 참고하세요.

📐 우리 네트워크 구성 (요약)

🍓 라즈베리파이 5 (Pi5) — LeKiwi 로봇
wlan0 → 10.10.1.10 (로봇망)
wlx..USB동글 → 192.168.50.110 (인터넷)
wg0 → 172.10.1.10 (VPN)
⬍ ASUS 라우터 (10.10.1.0/24 독립망)
💻 NUC (Control PC) — Phase 1 서버 설치 대상
eno1 유선 → 10.10.1.20 (로봇망)
wlp.. Wi-Fi → 192.168.50.190 (인터넷)
wg0 → 172.10.1.20 (VPN)
📌 네트워크 확인 명령어

NUC에서 확인 (Phase 1 서버가 설치될 곳)

# 1. IP 주소 확인 ip addr show | grep "inet " # ✅ eno1: 10.10.1.20 (로봇망), wlp...: 192.168.50.190 (인터넷) # 2. 로봇(Pi5)과 통신 확인 ping -c 2 10.10.1.10 # ✅ 0% packet loss → 네트워크 OK! # 3. 인터넷 확인 ping -c 2 8.8.8.8 # ✅ 0% packet loss → 인터넷 OK! # 4. 기본 라우팅 확인 ip route | grep default # ✅ default via 192.168.50.1 dev wlp... ← Wi-Fi가 인터넷 기본 경로 # (유선에 default가 없어야 합니다 → ipv4.never-default yes)
⚠️
핵심: DDS는 네트워크에 민감합니다!
인터페이스가 2개 이상이면 반드시 FastDDS XML로 화이트리스트 설정이 필요합니다. VPN(172.10.1.x)으로 ROS2 통신을 격리하는 것이 가장 안정적입니다. 상세 설정은 lekiwi-network-rtsp-edu.html의 STEP 3을 참고하세요.
🚨
네트워크가 안 되면 다음 단계로 절대 넘어가지 마세요!
ping 10.10.1.10이 실패하면 rosbridge, ROSboard, 등록 서버 모두 로봇 데이터를 받을 수 없습니다.
⚠️

⚠️ Ubuntu 24.04 사전 설정 (한 번만!)

🔒
24.04에서 달라진 점

Ubuntu 24.04는 Python 3.12를 사용하며, 시스템 Python에 pip install이 기본 차단됩니다.
아래 설정을 먼저 해두면 이후 작업이 수월합니다.

📌 사전 설정 — NUC에서 한 번만 실행
# 1. ROS2 Jazzy 자동 로드 (새 터미널마다 source 안 해도 됨) echo 'source /opt/ros/jazzy/setup.bash' >> ~/.bashrc # 2. pip install 허용 (Ubuntu 24.04 보안 정책 완화) mkdir -p ~/.config/pip cat > ~/.config/pip/pip.conf << 'EOF' [global] break-system-packages = true EOF # 3. 적용 확인 source ~/.bashrc ros2 --version # ✅ ros2 jazzy 나오면 성공 pip3 --version # ✅ pip 24.x 나오면 성공
B

B. rosbridge 설치 (DDS → WebSocket 변환기)

🌐
통역사에 비유하면

ROS2 로봇은 DDS라는 언어로 말합니다. 웹 브라우저는 WebSocket이라는 언어를 사용합니다.
rosbridge는 이 둘 사이에서 실시간 통역을 해주는 프로그램입니다.

📌 B-1 · rosbridge 설치

🖥️ NUC (터미널 1)

# rosbridge 서버 패키지 설치 # ros-jazzy-rosbridge-server = "ROS2 Jazzy용 rosbridge 서버" sudo apt update sudo apt install ros-jazzy-rosbridge-server -y # 설치 확인 (3개 패키지가 보여야 함) ros2 pkg list | grep rosbridge # ✅ rosbridge_server, rosbridge_library, rosbridge_msgs
📌 B-2 · rosbridge 실행
# rosbridge 실행 — 포트 9090에서 WebSocket 대기 ros2 launch rosbridge_server rosbridge_websocket_launch.xml # ✅ 이런 메시지가 나오면 성공: [rosbridge_websocket-1] Rosbridge WebSocket server started on port 9090 # ⚠️ 이 터미널은 계속 열어둡니다! (Ctrl+C 하면 종료됩니다)
📌 B-3 · 동작 확인

⭐ 새 터미널(터미널 2)을 열고 확인

# 포트가 열려있는지 확인 ss -tlnp | grep 9090 # ✅ LISTEN 0.0.0.0:9090 이면 정상 # HTTP로 접근하여 WebSocket 서버 확인 curl -s http://localhost:9090 # ✅ 'Can "Upgrade" only to "WebSocket".' → 정상!
🚨
"Address already in use" 오류? → 이미 rosbridge가 실행 중. pkill -f rosbridge로 종료 후 다시 실행
C

C. ROSboard 설치 (토픽 웹 뷰어)

📺
CCTV 관제실 모니터에 비유하면

rosbridge가 "통역사"라면, ROSboard는 CCTV 관제실의 모니터입니다.
카메라 영상, IMU 그래프, 배터리 게이지를 웹 브라우저에서 바로 볼 수 있습니다.

📌 C-1 · ROSboard 설치 (GitHub에서)

🖥️ NUC (터미널 2)

# ⚠️ ROSboard는 PyPI에 없습니다! GitHub에서 직접 설치해야 합니다. # (pip3 install rosboard 하면 "No matching distribution" 오류 발생) pip3 install git+https://github.com/dheera/rosboard.git # 설치 확인 pip3 show rosboard # ✅ Name: rosboard, Version: 1.3.x 이 나오면 성공
⚠️
사전 설정(pip.conf)을 안 했다면? 끝에 --break-system-packages를 붙여야 합니다
📌 C-2 · ROSboard 실행

새 터미널(터미널 3)을 열고 실행

# ⚠️ ros2 run rosboard는 안 됩니다! (GitHub pip 설치는 ROS2 패키지로 등록 안 됨) # Python 모듈로 직접 실행: python3 -m rosboard.rosboard --port 8888 # ✅ 이런 메시지가 나오면 성공: [INFO] ROSboard listening on :8888 # 💡 "simplejpeg not found" 경고는 무시해도 됩니다.
📌 C-3 · 브라우저에서 확인
# 브라우저에서 접속: http://10.10.1.20:8888 # 로봇망에서 http://192.168.50.190:8888 # 인터넷망에서 http://localhost:8888 # 같은 PC에서
📌 C-4 · 테스트 토픽으로 확인

새 터미널(터미널 4)에서 테스트 토픽 발행

# 테스트 토픽 발행 (rosbridge 또는 ROSboard 확인용) ros2 topic pub /test_battery sensor_msgs/msg/BatteryState \ "{voltage: 12.6, percentage: 0.85}" --rate 1 # ROSboard 웹에서 /test_battery 토픽이 보이면 성공! 🎉 # 클릭하면 실시간 값이 표시됩니다.

D. 등록 서버 실행 (FastAPI 대시보드)

🏨
호텔 프론트 데스크에 비유하면

ROSboard가 "CCTV 모니터"라면, 등록 서버는 호텔 프론트 데스크입니다.
로봇(손님)이 체크인하면 → 이름표를 달고 → CCTV(ROSboard)에 연결합니다.
rosbridge가 없어도 시뮬레이션 모드로 동작하여 대시보드를 미리 체험할 수 있습니다.

📌 D-1 · 의존성 설치
# 등록 서버에 필요한 Python 라이브러리 설치 pip3 install fastapi uvicorn roslibpy
📌 D-2 · 파일 배치
# 작업 폴더 생성 mkdir -p ~/phase1 cd ~/phase1 # 다운로드한 파일들을 이 위치에 배치:
~/phase1/ ├── phase1.html ← 대시보드 웹 페이지 (phase1 UI) ├── ros2_robot_register.py ← FastAPI 등록 서버 (메인 프로그램) └── requirements.txt ← Python 의존성 목록
📌 D-3 · 설정값 확인 (필요시 수정)

ros2_robot_register.py 파일 상단의 설정값을 본인 환경에 맞게 확인합니다:

# ros2_robot_register.py 상단 설정값 ROSBRIDGE_HOST = "localhost" # ← rosbridge가 같은 NUC이면 localhost ROSBRIDGE_PORT = 9090 # ← rosbridge 포트 DASHBOARD_PORT = 8080 # ← 대시보드 포트 ROSBOARD_URL = "http://10.10.1.20:8888" # ← ROSboard URL
📌 D-4 · 등록 서버 실행

새 터미널(터미널 5)에서 실행

# 작업 폴더로 이동 cd ~/phase1 # 등록 서버 실행 (순수 Python 프로그램 — ros2 명령이 아닙니다!) python3 ros2_robot_register.py # ✅ 이렇게 나오면 성공: ╔══════════════════════════════════════════════════╗ ║ 🏭 Phase 1 — 로봇 등록 서버 ║ ║ 📊 대시보드: http://0.0.0.0:8080 ║ ╚══════════════════════════════════════════════════╝
💡
rosbridge가 없어도 동작합니다! 등록 서버는 rosbridge 연결에 실패하면 자동으로 시뮬레이션 모드로 전환됩니다. 대시보드 UI를 미리 체험할 수 있습니다.

🔧 자주 나오는 문제와 해결법

🚨
localhost에서는 되는데 다른 IP로는 안 됨
→ 방화벽이 포트를 차단. sudo ufw allow 9090 && sudo ufw allow 8888 && sudo ufw allow 8080
🚨
rosbridge "Address already in use"
→ 기존 프로세스 종료 후 재실행: pkill -f rosbridge
🚨
ROSboard에서 토픽이 안 보임
→ ros2 topic list로 확인. /rosout만 보이면 로봇이 아직 연결 안 된 것. DDS Discovery + FastDDS XML 확인
🚨
ros2 run rosboard → "Package not found"
→ 정상! GitHub pip 설치는 ROS2 패키지로 등록 안 됨. python3 -m rosboard.rosboard 사용
🚨
DDS Discovery 간헐적 실패 (토픽이 보였다 사라짐)
→ 가장 흔한 원인: FastDDS 화이트리스트 미설정. FASTDDS_DEFAULT_PROFILES_FILE 환경변수 확인. 상세 내용은 nuc-dual-network-dds-troubleshooting.md 참고
🚨
ModuleNotFoundError: No module named 'fastapi'
→ pip3 install fastapi uvicorn roslibpy 실행. 사전 설정(pip.conf) 확인
