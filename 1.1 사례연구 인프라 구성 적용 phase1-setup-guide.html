2FastDDS 인터페이스 화이트리스트🍓 Pi5

Pi5에는 네트워크 인터페이스가 2개 있습니다. DDS 멀티캐스트가 양쪽으로 나가면 패킷 충돌이 발생하므로, 로봇망(10.10.1.x)만 사용하도록 제한합니다.

💡
왜 10.10.1.x만 지정하나요?

Pi5와 NUC 모두 IP가 2개입니다:

장치일반망 (인터넷)로봇망 (ASUS 독립)화이트리스트
🍓 Pi5192.168.50.110 (wlan0)10.10.1.10 (wlan1)10.10.1.10만 ✅
🖥️ NUC192.168.50.190 (wlan0)10.10.1.20 (eth0)10.10.1.20만 ✅

화이트리스트 없이 DDS를 실행하면 양쪽 인터페이스 모두에서 멀티캐스트가 나갑니다:
→ ❌ 일반망(192.168)에 로봇 제어 데이터 노출 (보안 위험)
→ ❌ 같은 일반망의 다른 ROS2 장비와 토픽 충돌
→ ❌ Discovery 패킷이 2배 → 네트워크 지연 증가

로봇망(10.10.1.x)만 지정 → Pi5↔NUC 전용 경로에서만 DDS 통신, 안전하고 빠름!
# Pi5의 네트워크 인터페이스 확인 (2개가 보여야 함)
ip -4 addr show | grep inet
# → inet 192.168.50.110/24  ← 일반망 (인터넷용)
# → inet 10.10.1.10/24     ← 로봇망 (ASUS) ★ 이것만 사용!

# FastDDS 화이트리스트 파일 생성
cat > ~/fastdds_wifi.xml << 'EOF'


  
    
      
        wifi_udp
        UDPv4
        
          
10.10.1.10
  <!-- ★ 로봇망 IP만! (192.168.x 제외) -->
        
      
    
    
      
        wifi_udp
        false
      
    
  

EOF

# 환경변수 설정 (bashrc에 추가)
echo 'export FASTDDS_DEFAULT_PROFILES_FILE=~/fastdds_wifi.xml' >> ~/.bashrc
source ~/.bashrc
⚠️
IP를 본인 환경에 맞게 변경! ip -4 addr에서 10.10.1.x 대역 IP를 확인 후 xml에 입력하세요.
3lerobot_bridge.py 배포 & 실행🍓 Pi5

lerobot_bridge.py는 Feetech 모터를 읽고/쓰며 ROS2 토픽을 발행하는 브릿지 노드입니다. Pi5에서 실행하면 NUC가 DDS로 토픽을 수신합니다.

# 로컬 PC → Pi5로 파일 전송

# Pi5에서 실행 (터미널 2)
source /opt/ros/jazzy/setup.bash
python3 ~/lerobot_bridge.py

정상 실행 시:

   모터: /dev/ttyACM0 @ 1000000, IDs=[6, 8, 9]
   발행: joint=50Hz, odom=10Hz, battery=1Hz
   구독: /lekiwi/cmd_vel → 역기구학 → 모터 속도 쓰기

발행 토픽 (Pi5 → DDS → NUC):

rosbridge는 Pi5에 없습니다! rosbridge(WebSocket 변환)는 NUC에서 실행합니다. Pi5는 lerobot_bridge(DDS 토픽 발행)만 담당합니다.

3. NUC — 기본 환경 설정
7FastDDS 인터페이스 화이트리스트🖥️ NUC

NUC도 Pi5와 마찬가지로 IP가 2개(일반망 + 로봇망)입니다. 로봇망(10.10.1.x)만 사용하도록 설정합니다.

# NUC에 SSH 접속
ssh bigman@10.10.1.20

# NUC 네트워크 인터페이스 확인 (2개)
ip -4 addr show | grep inet
# → inet 192.168.50.190/24  ← 일반망 (인터넷용)
# → inet 10.10.1.20/24     ← 로봇망 ★ 이것만 사용!

# FastDDS 화이트리스트 생성 (Pi5와 같은 원리, IP만 다름)
cat > ~/fastdds_wifi.xml << 'EOF'
<?xml version="1.0" encoding="UTF-8" ?>
<dds xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles">
  <profiles>
    <transport_descriptors>
      <transport_descriptor>
        <transport_id>wifi_udp</transport_id>
        <type>UDPv4</type>
        <interfaceWhiteList>
          <address>10.10.1.20</address>  <!-- ★ NUC 로봇망 IP만! -->
        </interfaceWhiteList>
      </transport_descriptor>
    </transport_descriptors>
    <participant profile_name="default_profile" is_default_profile="true">
      <rtps>
        <userTransports><transport_id>wifi_udp</transport_id></userTransports>
        <useBuiltinTransports>false</useBuiltinTransports>
      </rtps>
    </participant>
  </profiles>
</dds>
EOF

echo 'export FASTDDS_DEFAULT_PROFILES_FILE=~/fastdds_wifi.xml' >> ~/.bashrc
source ~/.bashrc
✅
Pi5(10.10.1.10)와 NUC(10.10.1.20) 모두 같은 10.10.1.x 대역에 화이트리스트
→ 이 대역에서만 DDS Discovery가 동작 → Pi5의 토픽을 NUC가 자동 발견합니다.
8rosbridge 설치 & 실행🖥️ NUC

rosbridge는 ROS2 DDS 토픽을 WebSocket으로 변환합니다. Pi5의 lerobot_bridge가 발행한 DDS 토픽을 NUC에서 수신하고, 웹 브라우저(대시보드)가 WebSocket으로 접근할 수 있게 합니다.

# rosbridge 설치
sudo apt install ros-jazzy-rosbridge-server -y

# 실행 (NUC 터미널 1)
source /opt/ros/jazzy/setup.bash
ros2 launch rosbridge_server rosbridge_websocket_launch.xml

# 정상 실행 확인
# → "Rosbridge WebSocket server started on port 9090"
✅
확인: ros2 topic list에서 Pi5 토픽이 보이면 DDS 연결 성공!
/lekiwi/joint_states, /lekiwi/odom, /lekiwi/battery가 보여야 합니다.
💡
데이터 흐름:
Pi5 lerobot_bridge.py → DDS(Wi-Fi) → NUC rosbridge → WebSocket(:9090) → 브라우저(대시보드)
4. NUC — MediaMTX 프록시 (WebRTC/HLS)

Pi5의 GStreamer RTSP 스트림을 받아서 WebRTC/HLS로 자동 변환합니다. NUC는 CPU 여유가 있으므로 프로토콜 변환 부담이 없습니다.

Pi5 → NUC 데이터 흐름: Pi5 GStreamer RTSP(:8554) /rgb ──→ NUC MediaMTX ──→ WebRTC :8889/rs_rgb ← 브라우저 최적! /depth ──→ NUC MediaMTX ──→ WebRTC :8889/rs_depth /usb ──→ NUC MediaMTX ──→ WebRTC :8889/cam_usb Pi5: 인코딩만 (CPU 가벼움) | NUC: 변환만 (CPU 여유)
9MediaMTX 설치🖥️ NUC
# NUC는 x86_64(amd64) → amd64 버전 다운로드
cd ~
wget https://github.com/bluenviron/mediamtx/releases/download/v1.16.1/mediamtx_v1.16.1_linux_amd64.tar.gz

# 압축 해제
tar xzvf mediamtx_v1.16.1_linux_amd64.tar.gz

# 실행 권한 확인
chmod +x ~/mediamtx
./mediamtx --help | head -3
🚨
NUC = amd64, Pi5 = arm64 — 아키텍처가 다릅니다!
NUC: mediamtx_v1.16.1_linux_amd64.tar.gz
Pi5: mediamtx_v1.16.1_linux_arm64.tar.gz
10프록시 설정 파일 생성🖥️ NUC
# Pi5 RTSP를 프록시하는 설정 파일 생성
cat > ~/mediamtx_proxy.yml << 'EOF'
# NUC MediaMTX 프록시 설정
# Pi5 GStreamer RTSP → WebRTC/HLS 변환

rtspAddress: :8564
hlsAddress: :8888
webrtcAddress: :8889

paths:
  rs_rgb:
    source: rtsp://10.10.1.10:8554/rgb
    sourceOnDemand: yes
  rs_depth:
    source: rtsp://10.10.1.10:8554/depth
    sourceOnDemand: yes
  cam_usb:
    source: rtsp://10.10.1.10:8554/usb
    sourceOnDemand: yes
EOF
⚠️
IP를 본인 Pi5 IP로 변경하세요. 현재: 로봇망 10.10.1.10, 일반망 192.168.50.110

설정 설명:

항목값설명
sourcertsp://Pi5:8554/...Pi5 GStreamer RTSP 원본
sourceOnDemandyes브라우저 접속 시에만 Pi5에 연결 (리소스 절약)
:8564RTSPVLC 등 전용 플레이어용
:8888HLS호환성 최대 (5~10초 지연)
:8889WebRTC브라우저 최적 (지연 1초 미만)
11MediaMTX 실행🖥️ NUC
# MediaMTX 프록시 실행 (NUC 터미널 2)
~/mediamtx ~/mediamtx_proxy.yml

브라우저에서 바로 확인:

프로토콜URL (NUC IP 기준)특징
🌐 WebRTChttp://10.10.1.20:8889/rs_rgb지연 1초 미만 (권장!)
🌐 WebRTChttp://10.10.1.20:8889/rs_depthRealSense Depth 컬러맵
🌐 WebRTChttp://10.10.1.20:8889/cam_usbUSB 카메라
📺 HLShttp://10.10.1.20:8888/rs_rgb/호환성 최대
✅
같은 WiFi의 모든 기기에서 접속 가능!
일반망 NUC IP가 192.168.50.190이면:
http://192.168.50.190:8889/rs_rgb 으로도 접속됩니다.

(선택) systemd 자동 시작:

# systemd 서비스 등록
sudo tee /etc/systemd/system/mediamtx.service << 'EOF'
[Unit]
Description=MediaMTX Proxy (Pi5 RTSP → WebRTC/HLS)
After=network-online.target
Wants=network-online.target

[Service]
User=bigman
ExecStart=/home/bigman/mediamtx /home/bigman/mediamtx_proxy.yml
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now mediamtx
