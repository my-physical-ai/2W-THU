🅰️ A팀 NUC 설정 예시

A팀 NUC — 유선 (RobotNet)
# 1. 유선 연결에 고정 IP 설정 (로봇 전용망) sudo nmcli con mod "유선 연결 1" \ ipv4.method manual \ ipv4.addresses 10.10.1.10/24 \ ipv4.never-default yes # 적용 sudo nmcli con down "유선 연결 1" && \ sudo nmcli con up "유선 연결 1"
A팀 NUC — Wi-Fi (zeta_amr_rmd)
# 2. Wi-Fi로 zeta_amr_rmd 연결 (AMR 통신) sudo nmcli dev wifi connect "zeta_amr_rmd" \ password "비밀번호" ifname wlp0s20f3 # 고정 IP 설정 (선택사항 — DHCP도 가능) sudo nmcli con mod "zeta_amr_rmd" \ ipv4.method manual \ ipv4.addresses 192.168.50.111/24 \ ipv4.gateway 192.168.50.1
⚠️
ipv4.never-default yes는 로봇망(유선)에 반드시 설정하세요! 이 설정이 없으면 인터넷 트래픽이 RobotNet으로 빠져서 인터넷이 끊깁니다.

🔒 팀별 ROS2 토픽 격리

4팀이 같은 RobotNet(10.10.1.x)을 공유하므로, ROS_DOMAIN_ID로 토픽을 격리해야 합니다.

📻
이해하기 쉽게!

무전기 채널에 비유하면 — A팀은 채널 0, B팀은 채널 1을 사용합니다. 같은 주파수(RobotNet)를 쓰지만 채널이 다르면 대화가 섞이지 않습니다.

💡 실제로는 — ROS_DOMAIN_ID가 다르면 DDS Discovery가 다른 도메인을 무시합니다. 토픽이 완전히 격리됩니다.

팀별 .bashrc 추가
# A팀의 모든 기기 (NUC, Pi5)에 추가: echo 'export ROS_DOMAIN_ID=0' >> ~/.bashrc # B팀: echo 'export ROS_DOMAIN_ID=1' >> ~/.bashrc # C팀: echo 'export ROS_DOMAIN_ID=2' >> ~/.bashrc # D팀: echo 'export ROS_DOMAIN_ID=3' >> ~/.bashrc # 적용 source ~/.bashrc

🔧 트러블슈팅

자주 발생하는 문제와 해결 방법입니다.

원인 1: AP 격리(Client Isolation) 활성화

ASUS 라우터의 AP 격리가 켜져 있으면 Wi-Fi 기기끼리 통신이 차단됩니다.

# 해결: ASUS 관리 페이지에서 AP 격리 해제 http://10.10.1.1 → 무선(Wireless) → 전문가 설정(Professional) → "AP 격리 사용" → 아니오(No) → 2.4GHz와 5GHz 둘 다 확인!

원인 2: Windows 방화벽

대상 기기가 Windows면 ICMP(ping)가 기본 차단됩니다.

# 관리자 권한 CMD에서 실행 (대상 기기에서!) netsh advfirewall firewall add rule \ name="ICMP Allow" protocol=icmpv4:8,any \ dir=in action=allow

NetworkManager 유선 프로필이 2개 이상 활성화된 경우입니다.

# 활성 프로필 확인 nmcli con show --active # 불필요한 프로필 삭제 sudo nmcli con delete "유선 연결 2"

ROS_DOMAIN_ID가 설정되지 않았거나 같은 값입니다.

# 현재 DOMAIN_ID 확인 echo $ROS_DOMAIN_ID # 팀별로 다르게 설정 (A=0, B=1, C=2, D=3) export ROS_DOMAIN_ID=0 # A팀 예시

ipv4.never-default yes 설정을 확인하세요.

# 라우팅 확인 ip route | grep default # default가 eno1(10.10.1.x)이면 문제! # 즉시 수정: sudo ip route del default via 10.10.1.1 sudo nmcli con mod "유선 연결 1" ipv4.never-default yes sudo nmcli con down "유선 연결 1" && sudo nmcli con up "유선 연결 1"

DDS 멀티캐스트 인터페이스가 잘못 선택된 경우입니다. FastDDS XML 화이트리스트를 설정하세요.

# FastDDS 인터페이스 화이트리스트 (환경변수 순서 중요!) export FASTDDS_DEFAULT_PROFILES_FILE=~/fastdds_wifi.xml # 1️⃣ 먼저 source /opt/ros/jazzy/setup.bash # 2️⃣ 나중에

🦾 RobotNet 확인

  • Pi5 Wi-Fi IP가 팀 번호와 일치 (10.10.1.2X)
  • 로봇 IP가 팀 번호와 일치 (10.10.1.3X)
  • ipv4.never-default yes 설정됨
  • AP 격리가 해제되어 있음

  • NUC Wi-Fi가 zeta_amr_rmd에 연결됨
  • IP가 192.168.50.11X와 일치
  • AMR 로봇과 ping 통신 확인

  • 학생 노트북이 NTH215에 연결됨
  • ping 8.8.8.8 성공
