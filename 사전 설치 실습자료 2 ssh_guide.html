📦 STEP 1

SSH 서버 설치

📞
비유

SSH 서버는 "원격 전화 수신 장치"예요. 이 장치가 없으면 다른 사람이 전화를 걸어도 받을 수 없어요. 지금 이 단계는 수신 장치를 설치하는 것입니다.

1

📋 패키지 목록 업데이트

최신 패키지 정보 가져오기

# 패키지 목록을 최신 상태로 갱신합니다
$ sudo apt update
▶ 이렇게 하면 — 마트 재고 목록을 새로 받아옵니다
  • 1
    인터넷에서 최신 패키지 목록을 다운로드합니다
  • 2
    Reading package lists... Done 가 나오면 성공입니다
📤 정상 출력
Hit:1 http://archive.ubuntu.com/ubuntu noble InRelease
Reading package lists... Done
2

📥 OpenSSH 서버 설치

SSH 수신 데몬 설치

# openssh-server 설치 (-y: 자동 Yes)
$ sudo apt install openssh-server -y
▶ 이렇게 하면 — 원격 전화 수신 장치가 설치됩니다
  • 1
    openssh-server 패키지를 다운로드합니다
  • 2
    설치 완료 후 SSH 서비스가 자동으로 시작됩니다
  • 3
    already the newest version 이 나오면 이미 설치된 것 — 정상!
📤 정상 출력
Reading package lists... Done
Building dependency tree... Done
Setting up openssh-server... Done
⚙️ STEP 2

SSH 서비스 시작 및 자동 시작 설정

3

🔍 SSH 서비스 상태 확인

제대로 실행 중인지 확인합니다

# SSH 서비스 상태 확인
$ sudo systemctl status ssh
▶ 이렇게 하면 — SSH 전화기가 켜져 있는지 확인합니다
  • 1
    서비스 상태 정보가 출력됩니다
  • 2
    active (running) 이 보이면 정상 실행 중입니다
  • 3
    inactive (dead) 이면 다음 명령으로 시작하세요
📤 정상 출력 (active 확인!)
● ssh.service - OpenBSD Secure Shell server
  Loaded: loaded (/lib/systemd/system/ssh.service)
  Active: active (running) since ...
4

🔁 자동 시작 등록 + 서비스 시작

재부팅 후에도 자동으로 실행되게 합니다

# 부팅 시 자동 시작 등록
$ sudo systemctl enable ssh
# 지금 바로 서비스 시작
$ sudo systemctl start ssh
▶ 이렇게 하면 — PC를 껐다 켜도 자동으로 SSH가 준비됩니다
  • 1
    enable — 시스템 부팅 시 자동 실행 목록에 등록합니다
  • 2
    start — 지금 이 순간 바로 SSH를 시작합니다
📤 enable 정상 출력
Synchronizing state of ssh.service...
Created symlink /etc/systemd/system/sshd.service
💡
3가지 systemctl 명령 차이
start — 지금 바로 시작  |  enable — 부팅 시 자동 시작 등록  |  restart — 재시작 (설정 변경 후 사용)
🔥 STEP 3

방화벽 설정 (UFW)

🏰
비유

방화벽(UFW)은 성문 경비병이에요. 방화벽이 켜져 있으면 허락된 문(포트)만 열려 있고 나머지는 모두 막혀 있어요. SSH는 22번 문을 사용하므로, 경비병에게 "22번 문은 통과시켜라"고 알려줘야 해요.

5

🔍 방화벽 상태 확인

UFW가 켜져 있는지 확인합니다

# 방화벽(UFW) 현재 상태 확인
$ sudo ufw status
▶ 이렇게 하면 — 성문 경비병이 서 있는지 확인합니다
  • 1
    Status: active → 방화벽이 켜져 있음 (SSH 허용 필요)
  • 2
    Status: inactive → 방화벽이 꺼져 있음 (SSH 바로 접속 가능)
6

✅ SSH 포트 허용

22번 포트를 방화벽에서 열어줍니다

# SSH(22번 포트) 통과 허용
$ sudo ufw allow ssh
# 방화벽 활성화 (꺼져 있을 경우)
$ sudo ufw enable
# 최종 상태 확인
$ sudo ufw status verbose
▶ 이렇게 하면 — 22번 문이 열립니다
  • 1
    allow ssh — 22번 포트를 허용 목록에 추가합니다
  • 2
    enable — 방화벽을 켭니다 (이미 켜져 있으면 생략)
  • 3
    verbose — 열린 포트 목록을 확인합니다
📤 정상 출력 (22번 허용 확인)
Status: active
22/tcp       ALLOW IN   Anywhere
22/tcp (v6)  ALLOW IN   Anywhere (v6)
🌐 STEP 4

내 IP 주소 확인

7

📡 IP 주소 확인

다른 PC에서 접속할 때 필요한 주소

# IP 주소 간단 확인
$ hostname -I
# 상세 네트워크 정보 (인터페이스별)
$ ip addr show
▶ 이렇게 하면 — 내 PC의 주소(전화번호)를 알 수 있습니다
  • 1
    hostname -I — 내 IP 주소를 짧게 출력합니다
  • 2
    보통 192.168.x.x 형태의 숫자가 내 주소예요
  • 3
    이 숫자를 다른 PC에서 접속할 때 사용합니다
📤 정상 출력 예시
192.168.1.100  # ← 이 주소를 기록해 두세요!
🔌 STEP 5

SSH 접속 테스트

⚠️
이 단계는 다른 PC(또는 같은 PC의 터미널 2개)에서 진행합니다.
같은 PC에서 테스트하려면 ssh localhost 또는 ssh 127.0.0.1을 사용하세요.
8

🔌 SSH 원격 접속

비밀번호로 첫 접속 테스트

# ssh [사용자명]@[서버 IP주소]
다른PC$ ssh zeta@192.168.1.100
# 같은 PC에서 테스트
$ ssh localhost
▶ 이렇게 하면 — 원격 전화 연결이 됩니다
  • 1
    처음 접속 시 "yes/no?" 가 나오면 yes를 입력합니다
  • 2
    비밀번호를 입력합니다 (화면에 표시 안 됨 — 정상!)
  • 3
    프롬프트가 zeta@서버이름:~$으로 바뀌면 접속 성공!
📤 처음 접속 시 (yes 입력!)
The authenticity of host '192.168.1.100' can't be established.
Are you sure you want to continue connecting (yes/no)? yes
zeta@192.168.1.100's password: (비밀번호 입력)
zeta@ubuntu-server:~$   ← 접속 성공!
💡
접속을 끊으려면 exit 또는 Ctrl + D를 누르세요.
🔑 STEP 6

SSH 키 생성 (비밀번호 없이 접속하기)

🏦
비유

지금까지는 비밀번호(열쇠 숫자)로 문을 열었어요. 이제부터는 열쇠 복사본(개인키)을 만들고, 자물쇠(공개키)를 서버에 달아두는 방식이에요. 내 열쇠를 가진 사람만 비밀번호 없이 들어올 수 있어요!

9

🔑 SSH 키 페어 생성

내 PC(클라이언트)에서 실행!

# ed25519 방식으로 키 생성 (최신 권장)
내PC$ ssh-keygen -t ed25519 -C "my-zeta-key"
▶ 이렇게 하면 — 개인키와 공개키 한 쌍이 만들어집니다
  • 1
    파일 저장 위치 물어보면 엔터 (기본값: ~/.ssh/id_ed25519)
  • 2
    passphrase 물어보면 엔터 2번 (비워두면 됩니다)
  • 3
    id_ed25519 = 개인키 (절대 유출 금지!), id_ed25519.pub = 공개키
📤 정상 출력 (엔터 3번이면 끝)
Generating public/private ed25519 key pair.
Enter file: [엔터]
Enter passphrase: [엔터]
Your public key saved in /home/zeta/.ssh/id_ed25519.pub
🚨
id_ed25519 (개인키)는 절대 다른 사람에게 보내거나 서버에 올리지 마세요!
공개키(.pub)만 서버에 등록합니다.
✅ STEP 7

서버에 공개키 등록

10

📤 공개키 서버에 복사

한 번만 등록하면 이후 비밀번호 없이 접속!

# 공개키를 서버에 자동 등록 (ssh-copy-id)
내PC$ ssh-copy-id zeta@192.168.1.100
▶ 이렇게 하면 — 서버에 자물쇠가 달립니다
  • 1
    마지막으로 비밀번호를 한 번 입력합니다
  • 2
    공개키가 서버의 ~/.ssh/authorized_keys에 저장됩니다
  • 3
    이후부터는 비밀번호 없이 바로 접속됩니다
📤 정상 출력
zeta@192.168.1.100's password: (마지막 비밀번호 입력)
Number of key(s) added: 1
Now try logging into the machine: 'ssh zeta@192.168.1.100'
11

🚀 비밀번호 없이 접속 확인

키 인증 최종 테스트

# 비밀번호 없이 바로 접속!
내PC$ ssh zeta@192.168.1.100
▶ 이렇게 하면 — 비밀번호 없이 바로 접속됩니다!
📤 키 인증 성공 출력
Welcome to Ubuntu 24.04 LTS
zeta@ubuntu-server:~$   ← 비밀번호 없이 바로 접속!
🔧 트러블슈팅

문제 해결

❌
Connection refused — SSH 서비스가 안 켜져 있어요
$ sudo systemctl start ssh
$ sudo ufw allow ssh
❌
Permission denied (publickey) — 공개키 권한 문제예요
서버$ chmod 700 ~/.ssh
서버$ chmod 600 ~/.ssh/authorized_keys
⚠️
REMOTE HOST IDENTIFICATION HAS CHANGED — 서버 재설치 후 발생
$ ssh-keygen -R 192.168.1.100
