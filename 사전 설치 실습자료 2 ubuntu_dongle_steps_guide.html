먼저 네트워크 상태를 보고, 그 결과에 따라 다음 명령을 정하는 방식 — 한 줄씩 순서대로 따라오세요

이 실습의 방식

한 번에 실행하는 스크립트가 아니에요. ① 명령 한 줄 입력 → ② 결과 보기 → ③ 결과에 따라 다음 단계 결정, 이걸 반복합니다.

①
STEP 1
nmcli 확인
②
STEP 2
상태 확인
③
STEP 3
현재 경로
④
STEP 4·5
동글 접속
⑤
STEP 6·7
충돌 해결
⑥
STEP 8
최종 점검
💡읽는 법: 각 단계마다 "▶ 이렇게 하면"(무슨 일이 일어나는지)과 "🔀 결과에 따라"(다음에 뭘 할지) 두 박스를 꼭 확인하세요.

단계별 실습

이 대화에서 NUC로 실제 진행한 순서 그대로입니다.

1🔍
nmcli 를 쓸 수 있나?도구가 있는지 먼저 확인
$which nmcli
▶ 이렇게 하면

네트워크 관리 도구(nmcli)가 깔려 있는지 알려줘요.

🔀 결과에 따라
/usr/bin/nmcli→경로가 나오면 사용 가능. STEP 2로 진행
(아무것도 없음)→아래 한 줄로 설치 후 STEP 2로
$sudo apt update && sudo apt install network-manager -y
2📋
네트워크 상태 알아보기가장 먼저 할 일
$nmcli device status
▶ 이렇게 하면

어떤 어댑터가 있고 무엇에 연결돼 있는지 전체 그림이 나와요.

wlp0s20f3 wifi 연결됨 NTH215 ← 내장 Wi-Fi wlx002e2d40106e wifi 연결 해제 -- ← USB 동글 eno1 ethernet 사용 불가 -- ← 유선
🔀 결과에 따라
동글 = 연결 해제→동글이 인식됨. STEP 3으로
동글이 안 보임→드라이버 문제. 동글을 다시 꽂거나 드라이버 설치
🏷️
쉽게 말하면

집에 있는 모든 출입문 목록을 펼쳐보는 거예요. 내장 문, 동글 문, 유선 문이 각각 어떤 상태인지 확인해요.

3🗺️
지금 인터넷이 어디로 나가나현재 경로 기록
$ip route
▶ 이렇게 하면

트래픽이 어느 길로 나가는지 보여줘요. 나중에 비교할 수 있게 꼭 봐둬요.

🔀 결과에 따라
default 1줄→인터넷 출구가 하나뿐. 정상
default 2줄+→출구가 여러 개라 충돌 가능. STEP 6·7 주의
💡default 로 시작하는 줄이 "인터넷으로 나가는 대문"이에요. 이 줄이 여러 개면 헷갈려서 느려질 수 있어요.
4📶
동글이 잡는 Wi-Fi 보기접속 전 목록 확인
$nmcli -f IN-USE,SSID,CHAN,RATE,SIGNAL device wifi list ifname wlx002e2d40106e
▶ 이렇게 하면

동글로 잡히는 Wi-Fi 목록과 신호·대역이 같이 나와요.

🔀 결과에 따라
SIGNAL 70+→신호 양호. 접속 진행
SIGNAL 50-→신호 약함. 동글 위치 조정
CHAN 36+→5GHz(빠름) / 1~13이면 2.4GHz(느림)
5🔗
동글로 Wi-Fi 접속접속 후 다시 상태 확인
$nmcli device wifi connect "동글SSID" password "비밀번호" ifname wlx002e2d40106e
▶ 이렇게 하면

동글로 그 Wi-Fi에 접속해요. "동글SSID" 와 "비밀번호" 는 실제 값으로 바꿔요.

1ifname 으로 "동글로 접속하라"고 콕 집어줘요.
2접속 후 아래 명령으로 상태를 다시 확인해요.
$nmcli device status
🔀 결과에 따라
둘 다 연결됨→출구가 두 개. 느려지거나 충돌 가능. STEP 6으로
속도 정상→문제 없으면 STEP 8로 점검만
⚠️실제 이 대화에서도 이 지점 다음에 충돌·느림이 생겼어요. 그래서 STEP 6·7로 정리합니다.
6⚖️
느리면: 빠른 쪽 metric 50우선순위 부여
$nmcli connection show
▶ 이렇게 하면

먼저 저장된 연결 이름을 확인해요. metric은 연결 이름에 거니까요.

1목록에서 빠른 어댑터(동글)의 연결 이름을 확인해요.
2그 연결에 metric 50을 줘요. (작을수록 우선)
$nmcli connection modify "동글SSID" ipv4.route-metric 50
$nmcli connection up "동글SSID"
🔀 결과에 따라 (ip route 로 확인)
metric 50 위로→빠른 쪽이 주력이 됨. 빨라졌으면 STEP 8로
그래도 느림→STEP 7로 (default 제외)
🎟️
쉽게 말하면

번호표예요. 50번이 200번보다 먼저 인터넷을 써요. 빠른 어댑터에 작은 번호를 준 거죠.

7🚦
그래도 느리면: 보조 default 제외이 대화 최종 해결
$nmcli connection modify "NTH215" ipv4.never-default yes
▶ 이렇게 하면

보조 어댑터를 인터넷 출구에서 빼요. (IP는 그대로 유지) "NTH215" 는 보조로 둘 연결 이름이에요.

$nmcli connection up "NTH215"
1metric 50만으로 부족할 때, 한쪽을 아예 default에서 빼면 깔끔해요.
2뺀 어댑터도 IP는 받고 있어서, 로봇·카메라 같은 특정 장비 통신은 계속 돼요.
3실제로 "하나만 default로" 하니 충돌이 사라지고 정상화됐어요.
🔀 결과에 따라 (ip route 로 확인)
default 1줄만→"하나만 default" 상태. 충돌 사라짐. STEP 8로
💡실전 팁: 동글은 인터넷용, 내장은 로봇(192.168.50.x)·카메라(192.168.1.x) 전용으로 나누면 깔끔해요. 이때 내장을 default에서 빼면 됩니다.
8✅
최종 점검실제로 어디로 나가나
$ip route get 8.8.8.8
▶ 이렇게 하면

인터넷이 실제로 어느 어댑터로 나가는지 콕 집어서 알려줘요.

🔀 결과에 따라
dev wlx... / wlp0s20f3→의도한 빠른 어댑터면 설정 성공!
엉뚱한 어댑터→STEP 6·7 다시 점검
💡재부팅 후에도 유지되는지 보려면 reboot 후 다시 ip route 로 확인해요. nmcli에 저장한 설정은 보통 영구 적용돼요.
