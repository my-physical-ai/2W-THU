실습 가이드 LeKiwi 원격조정 2

NUC를 X11으로 맞추고, 한글·pygame 문제를 해결한 뒤
조이스틱으로 LeKiwi를 움직이기까지 — 실제로 막혔던 곳을 순서대로 풉니다.

📌 이 실습자료는 무엇인가요?

1편에서 키보드/조이스틱 실행까지 했다면, 2편은 "실행했더니 안 되더라"를 다룹니다. NUC를 처음 세팅할 때 자주 막히는 다섯 고비 — ① 키보드가 글자만 찍힘(X11 문제), ② X11 바꿨더니 한글 안 됨, ③ 조이스틱이 pygame 없다고 함, ④ 호스트에 연결 안 됨, ⑤ 스틱 움직여도 로봇이 안 감 — 을 실제 겪은 순서대로 정리했습니다.

지금 쓰는 화면이 x11인지 wayland인지 한 단어로 알려줘요.

1x11이 나오면 → 키보드 조정이 됩니다. 3번으로 넘어가세요.
2wayland가 나오면 → 키보드 입력을 못 받아요. 2번으로 가서 바꿔야 합니다.
🔌
왜 중요한가요?

키보드를 읽는 도구(pynput)는 X11에서만 작동해요. Wayland에서는 키를 눌러도 프로그램이 못 받고, 글자가 터미널에 그냥 찍히기만 해요(waAWZ처럼). 그게 "안 움직이는" 진짜 원인이었어요.

2 🔄
Wayland → X11 전환wayland가 나왔을 때만
$sudo nano /etc/gdm3/custom.conf
▶ 이렇게 하면

로그인 화면 설정 파일을 열어 Wayland를 끄도록 바꿔요.

1파일 안에서 #WaylandEnable=true 줄을 찾아요.
2앞의 #을 지우고 true를 false로 바꿔요 → WaylandEnable=false
3Ctrl+O → Enter(저장) → Ctrl+X(닫기)
4sudo reboot로 재부팅한 뒤, 1번 명령으로 x11이 나오는지 다시 확인해요.
⚠️ 주의: X11으로 바꾸면 입력기(fcitx)가 자동 실행에서 풀려서 한글이 안 될 수 있어요. 그건 바로 다음 3번에서 고칩니다.
3 🇰🇷
한글 입력 되살리기X11 전환 후 한글이 안 될 때
$fcitx -r -d
▶ 이렇게 하면

꺼져 있던 한글 입력기(fcitx)를 다시 켜요. 보통 바로 한글이 됩니다.

1-r은 재시작, -d는 백그라운드 실행이에요.
2실행 후 편집기나 입력창에서 한/영 키를 눌러 한글이 써지는지 확인해요.
증상한글이 안 써진다 (X11으로 바꾼 직후)
원인환경변수(GTK_IM_MODULE=fcitx)는 남았지만, fcitx 프로그램 자체가 안 떠 있음
해결fcitx -r -d로 데몬을 다시 띄우면 됨 (한글 엔진은 이미 설치돼 있음)
💡 재부팅해도 자동으로 뜨게: 매번 손으로 안 켜도 되게 아래 한 줄을 등록해 두세요.
$echo 'fcitx &' >> ~/.xprofile
4 📦
조이스틱 라이브러리(pygame) 준비"No module named pygame" 오류 해결
$pip install pygame
▶ 이렇게 하면

조이스틱을 읽는 데 필요한 pygame을 설치해요.

1프롬프트 앞에 (lerobot)이 보이는 상태에서 설치하세요. (그 환경에 깔려야 함)
2설치 확인: pip list | grep pygame → pygame 2.x.x가 나오면 성공.
증상네트워크가 접근 불가능합니다 / pypi.org 연결 실패
원인NUC가 로봇 내부망(192.168.50.x)에만 연결돼 있어 인터넷이 안 닿음
해결설치할 때만 인터넷 되는 와이파이로 잠깐 전환 → 설치 후 다시 로봇망으로 복귀
🌐 인터넷 확인: ping 8.8.8.8이 되면 인터넷은 닿는 것(DNS 문제일 수 있음). 아예 안 되면 인터넷 되는 망으로 바꿔서 설치하세요. pygame은 한 번만 설치하면 됩니다.

스틱을 움직이면 어떤 axis 숫자가 변하는지 실시간으로 보여줘요.

실행 파일: 📄 axis_test.py (이 자료에 포함)

1왼쪽 스틱을 위아래로 밀어 어떤 axis가 변하는지 봐요 (전후 축).
2왼쪽 스틱을 좌우로 → 횡이동 축 확인.

4기본 매핑은 전후=axis1, 횡이동=axis0, 회전=axis3. 다르면 teleop_joystick_2.py의 축 번호를 고쳐요.
🛑 중요 — 코드는 터미널에 붙여넣지 마세요! 파이썬 코드를 터미널에 그냥 붙이면 bash가 한 줄씩 명령어로 읽어 깨집니다. 반드시 파일로 저장해서 python axis_test.py로 실행하세요. (이 자료의 axis_test.py를 쓰거나, nano axis_test.py로 만들기)

6 🎮
조이스틱으로 LeKiwi 움직이기호스트가 켜진 상태에서
$python teleop_joystick_2.py
▶ 이렇게 하면

스틱을 기울인 만큼 라즈베리파이의 LeKiwi 바퀴가 부드럽게 움직여요.

실행 파일: 📄 teleop_joystick_2.py

🕹️왼쪽 스틱 상하 axis 1
전진 / 후진 (위로 밀면 앞으로)
🕹️왼쪽 스틱 좌우 axis 0
왼쪽 / 오른쪽 옆걸음 (횡이동)
🕹️오른쪽 스틱 좌우 axis 3
제자리에서 좌/우 회전
1실행 전 조이스틱을 USB에 먼저 꽂아요. "Joystick Connected"가 뜨면 인식 성공.
2✅ Connected.가 떠야 호스트와 연결된 거예요. 안 뜨면 라즈베리파이 호스트부터 확인.
3스틱을 끝까지 확실히 밀어요. 살짝 기울이면 데드존(0.15)으로 무시돼요.
4끝낼 때는 Ctrl+C. 멈춤 신호를 보내고 깔끔히 종료해요.
증상Timeout waiting for LeKiwi Host to connect expired
원인라즈베리파이의 lekiwi_host가 안 켜져 있음 (SSH 끊기며 같이 죽는 경우 많음)
해결SSH 재접속 → ps aux | grep lekiwi_host 확인 → 죽었으면 호스트 다시 실행 후 클라이언트 실행
⚠️ 연결은 됐는데 안 움직일 때: ① 스틱을 끝까지 밀었는지(데드존), ② 축 번호가 맞는지(5번 axis_test.py로 확인), ③ 호스트 창에 에러가 없는지 순서로 확인하세요.
🛑 안전: 방향이 익숙해질 때까지는 바퀴를 받침에 올려 띄운 상태로 연습하세요. 책상에서 떨어지거나 케이블이 당겨질 수 있어요.

✅ 마무리 체크리스트

아래가 모두 ✅면 조이스틱 원격조정 준비 끝입니다.

  • X11 확인: echo $XDG_SESSION_TYPE → x11
  • 한글 입력: 편집기에서 한/영 키로 한글이 써짐 (fcitx 자동 시작 등록 완료)
  • pygame: pip list | grep pygame → 버전 표시됨
  • 조이스틱 인식: 실행 시 "Joystick Connected" 메시지
  • 호스트 연결: 라즈베리파이 호스트 켜짐 + 클라이언트에 ✅ Connected.
  • 축 매핑: axis_test.py로 확인한 번호가 코드와 일치
  • 안전: 첫 테스트는 바퀴를 띄운 상태로
🧭 실행 순서 기억: 라즈베리파이 호스트 먼저 → NUC에서 조이스틱 파일 실행. 받는 쪽(호스트)이 먼저 대기해야 거는 쪽(클라이언트)이 연결됩니다.
