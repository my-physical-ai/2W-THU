실습 가이드 FLIR ONE 열화상 카메라

USB로 꽂은 FLIR ONE을 Ubuntu에서 진짜 열화상 영상으로 띄우기까지, 한 단계씩 따라 합니다.

먼저 이것부터 이해해요 🧠

왜 FLIR ONE은 꽂아도 화면이 안 나올까요? 이걸 알면 모든 단계가 쉬워집니다.

🔌 FLIR ONE은 "일반 웹캠"이 아니에요

USB에 꽂으면 컴퓨터가 장치를 알아봅니다(lsusb에 보임). 하지만 일반 웹캠처럼 /dev/video0이 저절로 생기지 않아요. FLIR ONE은 자기만의 방식(전용 USB 프로토콜)으로 열화상 데이터를 주기 때문이에요.

🎥 일반 웹캠 (예: 로지텍)
USB → /dev/video0 자동 생성 → 바로 사용
🌡️ FLIR ONE
USB → libusb로 직접 읽기 → 가상 카메라로 변환
📮
쉽게 말하면

일반 웹캠은 "한국말"로 말해서 컴퓨터가 바로 알아들어요. FLIR ONE은 "자기만의 암호"로 말해서, 통역기(드라이버)를 거쳐야 컴퓨터가 알아듣습니다. 우리가 할 일이 바로 그 통역기를 설치하는 거예요.

단계별 실습 명령어 ⌨️

위에서 아래로, 1번부터 차례대로 따라 하세요. 각 명령은 복사 버튼으로 바로 복사할 수 있어요.

0 🔎
먼저 시리얼(식별번호) 찾기lsusb 전체 목록에서 FLIR 번호를 눈으로 확인
$lsusb
▶ 이렇게 하면

컴퓨터에 꽂힌 모든 USB 장치가 나와요. 그중 FLIR 줄의 번호를 찾는 게 첫 번째 할 일이에요.

1목록에서 이런 줄을 찾아요: Bus 001 Device 010: ID 09cb:1996 FLIR Systems FLIR ONE Camera
2가운데 ID 09cb:1996 이 부분이 시리얼(식별번호)이에요. 09cb는 제조사, 1996은 제품 번호.
3이 번호가 보이면 다음 칸(0-B)에서 그 번호를 그대로 넣어요. 안 보이면 USB를 다시 꽂아요.
📝중요: 시리얼 번호(09cb:1996)가 먼저 나와야 다음 명령에 넣을 수 있어요. 만약 다른 번호가 나오면(다른 FLIR 모델), 그 번호를 대신 사용하세요.
🪪
쉽게 말하면

lsusb는 교실에서 출석을 부르는 거예요. FLIR이 "저 여기 있어요(09cb:1996)"라고 대답하면, 그 학번(시리얼)을 받아 적어 두는 단계예요.

0 🔍
찾은 시리얼 넣어 자세히 확인그 번호로 FLIR만 콕 집어 진단
$lsusb -d 09cb:1996
▶ 이렇게 하면

방금 찾은 시리얼(09cb:1996)을 -d 뒤에 넣어, FLIR 하나만 콕 집어 확인해요.

1-d 09cb:1996의 09cb:1996이 바로 0-A에서 찾은 그 번호예요. 다른 번호였다면 바꿔 넣어요.
2FLIR Systems FLIR ONE Camera 한 줄만 나오면 USB 연결·전원·인식 모두 정상.
3이어서 ls /dev/video* → "없음"이 나와도 정상이에요 (FLIR은 웹캠이 아니니까요).
4sudo dmesg | grep -i flir로 커널이 FLIR을 봤는지 로그도 확인해요.
ℹ️참고: Cannot open device /dev/video0 오류가 떠도 당황하지 마세요. FLIR은 원래 /dev/video0을 안 만들어요. 다음 단계로 가면 됩니다.
1 🐍
파이썬으로 USB 접근 확인libusb로 직접 잡히는지 테스트
$conda activate lerobot && pip install pyusb libusb1
▶ 이렇게 하면

파이썬이 FLIR과 직접 대화할 수 있는 도구(pyusb)를 설치해요.

1설치 후 함께 드린 test_flir.py를 실행해요: python3 test_flir.py --detail
2FLIR ONE 발견!이 나오면 libusb로 접근 가능하다는 뜻이에요.
3권한 오류가 나면 sudo python3 test_flir.py로 다시 실행해요.
🤝
쉽게 말하면

Interface 0/1/2에 IN/OUT 통로가 보이면, FLIR이 "여기로 열화상 데이터 줄게"라고 약속한 거예요. 이제 그 데이터를 받아올 통역기만 켜면 됩니다.

2 📦
드라이버 빌드flirone-v4l2 내려받아 컴파일
$sudo apt install -y git cmake build-essential libusb-1.0-0-dev python3-opencv libjpeg-dev
▶ 이렇게 하면

FLIR 통역기를 만들 재료(빌드 도구·라이브러리)를 모두 준비해요.

1소스를 내려받아요: cd ~ && git clone https://github.com/fnoop/flirone-v4l2.git
2폴더로 들어가 컴파일해요: cd ~/flirone-v4l2 && make
3jpeglib.h 오류가 나면 sudo apt install -y libjpeg-dev 후 make clean && make 다시.
💡팁: make가 성공하면 폴더 안에 flirone 실행 파일이 생겨요. 이게 우리가 만든 "통역기"입니다.
3 📺
가상 카메라 만들기v4l2loopback으로 /dev/video2 생성
$sudo modprobe v4l2loopback devices=2 video_nr=2,3 card_label="FLIR_THERMAL","FLIR_VISIBLE" exclusive_caps=1
▶ 이렇게 하면

진짜 웹캠처럼 보이는 가짜 카메라 칸(/dev/video2, video3)을 만들어요.

1먼저 설치: sudo apt install -y v4l2loopback-dkms v4l2loopback-utils
2통역기가 열화상 데이터를 이 가상 칸에 흘려보내면, 다른 프로그램은 그냥 웹캠처럼 읽어요.
3확인: ls /dev/video* → /dev/video2, /dev/video3이 보이면 성공.
🚰
쉽게 말하면

FLIR(우물물)을 그대로는 못 마셔요. 가상 카메라는 수도꼭지 같은 거예요. 통역기가 우물물을 수도관에 부어주면, 우리는 수도꼭지(/dev/video2)에서 편하게 받아 씁니다.

4 🔥
열화상 실행 & 화면 보기터미널 2개로 캡처 + 재생
$sudo ./flirone ./palettes/Iron_Black.raw
▶ 이렇게 하면

FLIR이 찍은 열화상이 가상 카메라로 흘러가기 시작해요.

1터미널 1에서 위 명령으로 캡처를 시작해요 (cd ~/flirone-v4l2 안에서).
2터미널 2(새 창)에서 화면을 봐요: ffplay /dev/video2
3드디어 따뜻한 곳은 밝게, 차가운 곳은 어둡게 보이는 열화상이 떠요! 🎉
⚠️자주 나는 오류: Assertion 'fdwr1 >= 0' failed가 뜨면, 3번 가상 카메라(/dev/video2)가 안 만들어진 거예요. 3번 단계를 먼저 실행하세요.
