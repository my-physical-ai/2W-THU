모니터링 화면 만들기
템플릿을 골라 우리 팀 스타일로

카메라 서버는 그대로 두고, NUC에서 Flask 모니터링 화면을 띄웁니다. 관제센터형·카드형 두 템플릿을 출발점으로 팀이 디자인을 입히고, 완성한 화면을 폰이나 PC로 발표해요.

🖥️ 백엔드: NUC (공통) 🎨 화면: 템플릿 2종 (팀이 꾸밈) 📱 발표: 폰·PC 접속

🎯 이 실습의 목표 — 프로젝트 과제

정해진 정답을 따라 만드는 게 아니라, 템플릿을 골라 팀만의 화면을 완성하는 프로젝트예요.

📌 무엇을 건드리고, 무엇을 건드리지 않나
  • 그대로 둠 (건드리지 않음): 라즈베리파이의 카메라 서버(camera_server_pi5_v2.py), 기존 NUC 수신, IP 카메라 연결 — 전부 유지
  • 새로 실행: NUC에서 Flask 모니터링 서버(nuc_monitor_server.py) 하나 — 카메라 영상을 모아 화면으로 제공
  • 팀이 꾸밈: templates/ 폴더의 화면 HTML 2종 — 색·배치·기능을 자유롭게 디자인

⚙️ 먼저 이해하기 — "서버 켜기"와 "화면 보기"는 다른 일

터미널에서 실행하면 영상 창이 뜨는 게 아니에요. 헷갈리기 쉬우니 먼저 짚고 가요.

🤔 "실행했는데 아무것도 안 나와요"

python3 nuc_monitor_server.py 는 서버를 켜는 명령이에요. 창에 영상이 뜨는 게 아니라, "이제 접속을 받을 준비 완료" 상태로 터미널이 계속 떠 있는 게 정상이에요. 화면은 브라우저로 따로 봅니다.

💡 "인덱스(index.html)는 백에서 도는 건가요?" → 맞아요. index.html은 더블클릭해 여는 파일이 아니라, 서버가 떠 있을 때 브라우저가 /로 요청하면 서버가 내려주는 화면이에요. 영상(/stream/...)도 서버가 살아 있어야 흘러나와요. 서버가 꺼져 있으면 화면을 열어도 영상이 안 나옵니다.
⚠️ 주의: 서버를 켠 터미널 창을 닫으면 서버도 꺼져요. 발표하는 동안에는 터미널을 계속 열어둔 채 브라우저로 보세요.

📁 파일 구조 — 어디를 고치나

백엔드는 1개, 화면 템플릿은 templates/ 안에 있어요. 팀은 노란색만 고치면 돼요.

nuc_monitor_server.py # 백엔드 (공통) — 고치지 않음 templates/ ├── index.html # 템플릿 고르는 시작 페이지 (선택사항) ├── template_control.html # 템플릿 1 · 관제센터형 ← 팀이 꾸밈 └── template_cards.html # 템플릿 2 · 카드형 ← 팀이 꾸밈
🧩 템플릿 안의 [TEAM] 주석을 찾으면, 색·배치·기능을 어디서 고치는지 표시돼 있어요.

🎨 두 가지 템플릿 — 공장 운영 화면 샘플

실제 공장처럼 꾸민 샘플이에요. 진짜 영상이 나오는 카메라는 3대(웹캠2+IP1)이고, 나머지 구역(B라인·포장·창고 등)은 "공장엔 카메라가 많다"를 보여주는 데모 셀이에요. 팀이 이 형태에서 출발해 우리 공장에 맞게 꾸밉니다.

🛰️ 관제센터형
큰 메인 1개 + 사이드 썸네일
  • 구역별 카메라 목록(입구·A라인·B라인·포장·창고)
  • 큰 메인 + 멀티 썸네일 + 상태 패널·이벤트 로그
  • 한 대를 크게 강조하며 설명하기 좋음
  • 실제 카메라 3대는 영상, 나머지는 데모 셀
접속: /control
🗂️ 카드형
동등한 카드 격자
  • 생산 라인 / 품질 검증 / 물류 그룹으로 묶음
  • 카드마다 KPI(가동률·불량률·생산량 등) 표시
  • 3D 느낌 — 마우스 올리면 카드가 기울며 떠오름
  • 스캔 라인·상태 태그로 공장 느낌
접속: /cards
💡 고르는 기준: "한 대를 크게 보여주며 설명" → 관제센터형. "여러 대를 한눈에 비교" → 카드형. 둘 다 꾸며보고 발표에 맞는 쪽을 골라도 좋아요.

2영상은 여기 터미널이 아니라 브라우저에서 봐요. 창은 끄지 말고 두세요.

시작 페이지에서 관제센터형/카드형을 골라 들어가요. 둘 다 반응형이라 폰에서도 깔끔해요.

1/ 로 들어가면 두 템플릿을 고르는 화면이 나와요.
2/control 또는 /cards 로 바로 들어갈 수도 있어요.
3"📸 스냅샷"을 누르면 그 카메라 화면 1장이 NUC에 저장돼요.
📶 폰 접속 조건: 폰이 NUC와 같은 네트워크여야 하고, NUC 방화벽이 8080을 막지 않아야 해요.
4 🎨
템플릿 꾸미기templates/ 안의 HTML
▶ 이렇게 하면

템플릿 HTML의 [TEAM] 자리를 고치면 화면이 우리 팀 색으로 바뀌어요.

# 예) 관제센터형 색 테마 — template_control.html 위쪽 :root
:root{
  --bg:#0a0e1a;      /* 전체 배경색 — 여기만 바꿔도 분위기가 확 달라져요 */
  --accent:#22d3ee;  /* 강조색 (관제 시안) */
}
1파일을 저장하고 브라우저를 새로고침하면 바로 반영돼요. (서버 재시작 불필요)
2색 → 배치 → 기능 순으로 조금씩 바꿔보면 안전해요.
⚠️ 주의: 영상은 /stream/카메라이름, 목록은 /api/cameras 로 들어와요. 이 주소들은 바꾸지 말고 그대로 써야 영상이 나와요.

📱💻 폰이나 PC로 발표하기

완성한 화면을 폰이나 PC로 띄워 팀이 발표해요. 둘 다 같은 주소로 접속하면 돼요.

5 🎤
발표 준비폰·PC에서 한 번 더 점검
1발표할 기기(폰 또는 PC)를 NUC와 같은 네트워크에 연결해요.
2브라우저에 http://<NUC IP>:8080/control(또는 /cards)을 띄워요.
3PC·빔프로젝터로 발표하면 큰 화면으로, 폰이면 가로로 돌려 시원하게 보여줘요. 발표 중 썸네일/필터로 카메라를 바꿔가며 설명해요.
4미리 즐겨찾기에 주소를 저장해 두면 발표 때 빠르게 열 수 있어요.
💡 발표 팁: 영상이 끊기면 당황하지 말고 새로고침. 백엔드가 자동 재접속하므로 잠시 뒤 다시 나와요. 두 화면 다 반응형이라 PC·폰 어디서 열어도 깔끔해요.

✅ 추천 진행 순서

1. Pi5에서 camera_server_pi5_v2.py 실행 (웹캠 2개 송출, 기존 그대로)
2. NUC에 pip install flask opencv-python
3. nuc_monitor_server.py 의 CAMERAS를 우리 IP/주소로 확인
4. python3 nuc_monitor_server.py 실행 → 주소 확인
5. PC·폰에서 /control 과 /cards 둘 다 열어보기
6. 팀이 쓸 템플릿 하나 선택
7. [TEAM] 자리를 고쳐 우리 팀 스타일로 디자인
8. 폰이나 PC로 띄워 발표 리허설
